### 3. 问题解决

- 遇到的问题：aiPolish 云函数调用 TokenHub AI API 时，API Key 如果硬编码在代码中存在安全风险，且不方便管理
- 解决方案：将 API Key 通过微信云开发控制台的环境变量功能配置为 `TOKENHUB_API_KEY`，云函数中通过 `process.env.TOKENHUB_API_KEY` 读取，实现密钥与代码分离，提升安全性

- 遇到的问题：health 云函数部署后数据库检查失败，返回 `database: disconnected`
- 解决方案：排查发现数据库集合名写成了 `users`，实际集合名为 `user`，修正后数据库检查恢复正常

## 心得体会

本次云服务部署让我深入理解了微信云开发的架构。项目使用腾讯云作为底层基础设施，通过微信云开发平台统一管理云函数、数据库和存储，省去了传统云服务部署中配置服务器、Nginx 反向代理、SSL 证书等繁琐步骤。

在环境变量配置过程中，我认识到密钥管理的重要性。将 API Key 从代码中抽离到环境变量，不仅避免了密钥泄露风险，也方便在不同环境（开发/生产）之间切换配置。

部署过程中也遇到了一些问题，比如数据库集合名称不一致导致健康检查失败，这提醒我在开发时要保持代码与实际数据结构的一致性，养成及时核对的好习惯。
