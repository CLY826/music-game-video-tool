## 心得体会

在 AI 集成过程中，我学到了如何将大模型 API 与小程序业务场景结合。aiPolish 云函数通过调用 TokenHub 的 hy3-preview 模型，根据用户输入的备注内容和歌曲信息，生成更自然、更有音游社区氛围的文案。

后端实现上，我设计了专门的 system prompt，约束 AI 保持用户原意、使用音游圈术语（如 FC、AP、收歌等）、控制字数在 50~150 字之间，确保润色结果符合社区调性。同时添加了鉴权校验、输入长度限制（200 字）、请求超时（15 秒）等保护措施，防止接口被滥用。

前端调用上，通过 wx.cloud.callFunction 调用云函数，设置 20 秒超时，并在润色过程中显示 loading 状态，润色完成后将结果回填到输入框，用户可以对比原文和润色后的内容。

安全方面，API Key 通过环境变量注入而非硬编码在代码中，避免了密钥泄露风险。错误处理覆盖了未授权调用、API Key 未配置、输入为空、输入超长、API 返回异常、请求超时等多种场景，保证用户体验友好。
