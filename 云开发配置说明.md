# 音游视频练习辅助小程序 - 云开发配置与数据库文档

---

## 一、快速启动流程

### Step 1：创建小程序

1. 打开 [微信公众平台](https://mp.weixin.qq.com)，注册/登录个人小程序账号（无需企业认证）
2. 记录你的 **AppID**，填入 `project.config.json` 的 `"appid"` 字段

### Step 2：开通云开发

1. 打开 **微信开发者工具**，导入本项目（选 `rhythm-practice` 文件夹）
2. 点击顶部工具栏「云开发」按钮
3. 第一次进入会提示开通环境 → 点击「新建环境」，名称随意，选 **免费版**
4. 等待约 1 分钟，记录自动生成的 **环境 ID**（格式如 `cloud-xxxxx-xxxxxx`）

### Step 3：填入环境 ID

修改以下两处，把 `YOUR_ENV_ID` 替换为你的实际环境 ID：

- `app.js` 第 12 行 `env: 'YOUR_ENV_ID'`
- `app.js` 第 45 行 `envId: 'YOUR_ENV_ID'`

### Step 4：初始化数据库集合

进入**云开发控制台 → 数据库**，依次点击「+」创建以下集合（名称必须完全一致）：

| 集合名 | 用途 |
|--------|------|
| `game` | 音游分类 |
| `video` | 视频信息 |
| `comment` | 评论 |
| `user` | 用户信息 |

> 首次运行小程序后，`game` 集合会由代码自动写入默认分类，无需手动录入。

### Step 5：设置数据库权限

进入每个集合的「权限设置」，按如下配置：

| 集合 | 权限模式 |
|------|----------|
| `game` | **所有人可读，仅创建者可写**（实际选"自定义安全规则"）|
| `video` | **所有人可读，仅创建者可写** |
| `comment` | **所有人可读，仅创建者可写** |
| `user` | **仅创建者可读写** |

更推荐在集合的「安全规则」设置中使用以下 JSON（以 `video` 为例）：
```json
{
  "read": true,
  "write": "auth.openid == doc.openid"
}
```

对于 `game` 集合（只读，由代码首次写入）：
```json
{
  "read": true,
  "write": "auth.openid != null"
}
```

### Step 6：部署云函数

在微信开发者工具中，右键点击 `cloudfunctions` 下的每个函数文件夹，选择：
- 「上传并部署：云端安装依赖（不上传 node_modules）」

需要部署的云函数：
- `login`
- `getVideos`
- `getComments`
- `addComment`

### Step 7：配置云存储

云存储无需手动配置，代码中已使用 `wx.cloud.uploadFile` 自动上传。
默认存储路径：
- 视频：`videos/{openid}/{timestamp}.mp4`
- 封面：`thumbs/{openid}/{timestamp}.jpg`

## 二、数据库集合字段说明

### `game` 集合 — 音游分类

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 自动生成 |
| `name` | String | 音游名称，如 `Phigros` |
| `icon` | String | Emoji 图标 |
| `color` | String | 主题色（十六进制），如 `#6c63ff` |
| `sort` | Number | 排序权重，越小越靠前 |
| `videoCount` | Number | 该分类下的视频数量（冗余字段，提升查询效率） |

**示例记录：**
```json
{
  "_id": "auto",
  "name": "Phigros",
  "icon": "🎵",
  "color": "#6c63ff",
  "sort": 0,
  "videoCount": 5
}
```

---

### `video` 集合 — 视频信息

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 自动生成 |
| `songName` | String | 歌曲名称 |
| `gameId` | String | 关联 `game._id` |
| `gameName` | String | 冗余音游名称（减少 JOIN） |
| `gameColor` | String | 冗余主题色 |
| `desc` | String | 练习说明/备注（可选） |
| `videoFileId` | String | 云存储 FileID，用于永久引用 |
| `videoUrl` | String | 临时访问 URL（有效期7天，需定期刷新） |
| `thumbFileId` | String | 封面图云存储 FileID |
| `thumbUrl` | String | 封面图临时 URL |
| `duration` | Number | 视频时长（秒） |
| `durationText` | String | 格式化时长，如 `02:35` |
| `fileSize` | Number | 文件大小（字节） |
| `openid` | String | 上传者 openid |
| `uploaderName` | String | 上传者昵称 |
| `uploaderAvatar` | String | 上传者头像 URL |
| `commentCount` | Number | 评论数（冗余，避免 COUNT 查询） |
| `createTime` | Date | 上传时间（`db.serverDate()`） |

---

### `comment` 集合 — 评论

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 自动生成 |
| `videoId` | String | 关联 `video._id` |
| `content` | String | 评论内容 |
| `openid` | String | 评论者 openid |
| `authorName` | String | 评论者昵称 |
| `authorAvatar` | String | 评论者头像 URL |
| `createTime` | Date | 评论时间 |

---

### `user` 集合 — 用户信息

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | String | 自动生成 |
| `openid` | String | 微信 openid（唯一标识） |
| `appid` | String | 小程序 appid |
| `nickName` | String | 用户昵称 |
| `avatarUrl` | String | 用户头像 URL |
| `createTime` | Date | 首次登录时间 |
| `lastLoginTime` | Date | 最近登录时间 |

## 四、Tab Bar 图标说明

项目中使用了以下图标路径（需自行准备 PNG 图标，尺寸建议 81×81px）：

```
images/
├── tab_home.png          首页-未选中
├── tab_home_active.png   首页-已选中
├── tab_community.png     社区-未选中
├── tab_community_active.png
├── tab_profile.png       个人中心-未选中
└── tab_profile_active.png
```

**快速解决方案：** 可从 [iconfont.cn](https://www.iconfont.cn/) 或 [Flaticon](https://www.flaticon.com/) 免费下载图标，导出为 PNG。
或者也可以临时去掉 `app.json` 中的 `iconPath` / `selectedIconPath` 字段，微信开发者工具会显示默认图标。

## 六、常见问题

**Q: 视频播放报错 `ERR_SRC_NOT_SUPPORTED`？**
A: 云存储的临时 URL 有效期为 **7天**，过期后需调用 `wx.cloud.getTempFileURL` 重新获取。播放页已自动处理此逻辑。

**Q: 上传视频很慢？**
A: 免费版云存储上行带宽有限，视频文件建议压缩到 50MB 以内。

**Q: 搜索不到结果？**
A: 云数据库正则查询区分数据库字段名，确保 `video` 集合中字段名为 `songName`（驼峰命名）。

**Q: 无法获取用户头像/昵称？**
A: 微信从 2022 年起不再允许直接获取用户信息，需使用 `<button open-type="getUserInfo">` 或新版头像选择组件。本项目已默认使用「匿名用户」处理未授权情况。
